# N4 QUANT データ自動更新パイプライン

ナンバーズ4の当せん番号を GitHub Actions で毎営業日自動取得し、GitHub Pages で `data.json` として公開する構成です。N4 QUANT Webアプリの「データソースURL」にこのJSONを設定すると、ログイン不要で「開くだけで最新データ」が実現します。

## 構成

```
n4-pipeline/
├── .github/workflows/update.yml   # 平日20:30(JST)に自動実行
├── scripts/fetch_n4.py            # 取得・マージスクリプト(依存ライブラリなし・標準ライブラリのみ)
├── docs/data.json                 # 公開されるデータ本体(GitHub Pagesのソース)
└── README.md
```

## セットアップ手順(約10分)

1. **リポジトリ作成**: GitHubで新規リポジトリ(例: `n4-data`)を **Public** で作成
   ※ Publicにするのは GitHub Pages を無料枠で使うため。当せん番号は公開情報なので問題ありません

2. **ファイル配置**: このフォルダの中身をそのままリポジトリのルートにアップロード(Web UIの "Add file > Upload files" でフォルダごとドラッグでOK)

3. **Actionsの書き込み許可**: リポジトリの Settings > Actions > General > Workflow permissions を **"Read and write permissions"** に設定

4. **Pages有効化**: Settings > Pages > Source を "Deploy from a branch"、Branch を `main` / `/docs` に設定して Save

5. **初回実行**: Actions タブ > "Update Numbers4 data" > "Run workflow" で手動実行。成功すると `docs/data.json` に直近の結果がコミットされます

6. **URL確認**: 数分後に以下のURLでJSONが見えることを確認
   ```
   https://<あなたのユーザー名>.github.io/<リポジトリ名>/data.json
   ```

7. **アプリに設定**: N4 QUANT Webアプリの「データ」タブ > データソースURL に上記URLを貼って保存 > 「↻ データ更新」

以降は平日20:30(JST)に自動更新され、アプリを開いて「↻ データ更新」を押すだけで最新化されます。

## 過去データのバックフィル

このパイプラインが取得するのは各ソースに掲載されている直近分のみです。過去の履歴をまとめて入れたい場合は、`docs/data.json` に手動で以下の形式の要素を追記してコミットしてください(アプリの一括インポート機能でローカルに直接入れても構いません)。

```json
[
 {"round": 6798, "date": "2026-07-08", "number": "0731"},
 {"round": 6799, "date": "2026-07-09", "number": "2214"}
]
```

## 注意事項(正直な制約)

- **HTML構造の変化で壊れる可能性があります。** スクレイパーはタグ除去+正規表現のゆるい抽出で構造変化にある程度耐えますが、大幅なリニューアルには追従できません。Actionsが失敗し始めたらログを確認してください(失敗時はexit 1で通知が飛びます)。取得0件のまま黙ってデータを壊すことはありません(既存データ優先マージ)。
- **アクセス頻度は1日1回です。** 個人利用の範囲に留め、cronの頻度を上げないでください。取得先サイトの利用規約もご自身でご確認ください。
- **データ検証**: 誤抽出防止として、既存の回号と番号が食い違う場合は既存を保持して警告のみ出します。まれな取りこぼしはアプリの手動追加で補正してください。
